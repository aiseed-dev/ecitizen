X-13ARIMA-SEATS の使い方

X-13ARIMA-SEATS は「spec ファイル」というテキストファイルに設定を書き、 コマンドラインで実行するだけで動きます。GUI はありませんが、 設定ファイルの基本形を1つ覚えれば使い回しできます。

1. spec ファイルを書く

月次の売上データを季節調整する最小限の例です。sales.spc という ファイル名で保存します (拡張子は .spc)。 

series{
  title = "月次売上"
  start = 2018.jan
  period = 12
  data = (
    5230 4890 5560 5820 6010 5770
    5480 5150 5900 6120 5980 7450
    5410 5020 5730 6010 6180 5940
    ...
  )
}
transform{ function = auto }   # 対数変換するかを自動判定
automdl{ }                     # ARIMAモデルを自動選択
outlier{ }                     # 外れ値を自動検出
x11{ }                         # X-11方式で季節調整

主なセクション (spec) の意味:

spec役割
seriesデータ本体。開始年月 (start)、周期 (period: 月次=12、四半期=4)、データ (data=(...) に直接書くか file="..." で別ファイル)
transform変換。function=auto にすると対数変換の要否を自動判定
regression曜日効果・祝日・介入などの回帰変数。例: variables=(td) で曜日調整
automdl回帰ARIMAモデルの自動選択 (旧X-12の pickmdl より高性能な TRAMO 系)
outlier外れ値 (AO/LS/TC) の自動検出
x11X-11方式の季節調整を行う
seatsSEATS方式 (モデルベース) の季節調整を行う。x11{} の代わりに書く

2. 実行する

x13as_html sales        # 拡張子 .spc は付けない
# → sales.html (結果)、sales_log.html、sales_err.html が出力される

x13as_ascii sales
# → sales.out、sales.log、sales.err (テキスト)

付属サンプルで試すこともできます。配布物の examples にある testairline.spc は国際航空旅客数の有名なデータです。

3. 結果を見る

X-11方式の主要な出力表は番号で呼ばれます。まずはこの3つを見れば十分です:

表内容
D11季節調整済み系列 (求めたかった結果)
D10季節要因 (毎年の季節パターン)
D12トレンド・サイクル

系列をファイルに保存したい場合は spec に保存指定を足します: 

x11{ save = (d11 d10 d12) }
# → sales.d11 などの表形式テキストが出力される

調整がうまくいっているかは、出力末尾の M統計量・Q統計量 (1未満なら良好) と、残差の季節性チェック (「no evidence of residual seasonality」と出れば OK) を確認します。

4. X-11方式と SEATS方式

X-13ARIMA-SEATS の名前のとおり、季節調整の方式は2つから選べます。 x11{ } を seats{ } に書き換えるだけで切り替えられます。

  • X-11方式: 移動平均に基づく伝統的な方式。日本の官庁統計の多くはこちら (X-12-ARIMA からの継続性がある)
  • SEATS方式: ARIMA モデルに基づく方式。欧州の統計局で広く使われる

どちらが良いかは系列によりますが、まず X-11 で始めて、 比較したくなったら SEATS も試す、で十分です。

5. X-12-ARIMA からの移行

X-12-ARIMA の spec ファイルはほぼそのまま X-13ARIMA-SEATS で動きます。 アーカイブの X-12-ARIMA 使用手順にある spec の書き方・出力表の読み方は基本的に共通です。

※ 日本のデータで曜日調整 (regression の td) を使う場合、祝日は アメリカの祝日が前提なので注意してください。日本の祝日効果を入れるには genhol ユーティリティ等で独自の回帰変数を作る必要があります (各府省の季節調整の資料が参考になります)。
